## 文件职责

| 文件 | 当前职责 |
| --- | --- |
| `cli.py` | 通过可注入 Console 接收输入、展示流式文本并处理终端命令 |
| `agent.py` | 管理多轮历史、压缩检查点、受限工具循环和修改后验证工作流 |
| `context.py` | 估算上下文、选择完整历史轮次并保存待应用的压缩结果 |
| `diagnostics.py` | 为 `/doctor` 执行不联网、不泄露密钥的本地环境检查 |
| `instructions.py` | 分层加载、热重载并非覆盖式初始化有界 `AGENTS.md` 项目指令 |
| `evals.py` | 运行默认离线评测和双重显式开启的真实 DeepSeek 验收 |
| `llm.py` | 封装 DeepSeek 请求、可观察重试、流式事件、工具调用和服务端用量解析 |
| `noninteractive.py` | 执行一次性只读请求并输出版本化 text/json/JSONL 协议 |
| `hooks.py` | 提供有界、类型化的模型与工具生命周期回调 |
| `audit.py` | 使用生命周期 hooks 写入元数据专用的有界本地 JSONL 审计 |
| `checkpoint.py` | 保存本进程 Agent 文件编辑的有界恢复栈和内容身份 |
| `errors.py` | 定义应用、Agent、LLM 和工具层的用户可见异常层级 |
| `schemas.py` | 定义消息、思考内容、工具定义、调用、结果和模型响应结构 |
| `task.py` | 管理单次任务计划、步骤状态和最近一次质量检查快照 |
| `session.py` | 迁移、命名、搜索、原子保存和恢复工作区内的版本化会话快照 |
| `config.py` | 从环境变量或 `.env` 读取并校验 API Key、模型、提示词、重试和运行限制 |
| `registry.py` | 注册工具、生成预览、执行审批检查并统一返回结果 |
| `filesystem.py` | 在工作区内读取、搜索以及原子写入 UTF-8 文件 |
| `shell.py` | 执行固定质量检查、只读 Git，以及需审批的明确路径暂存和本地提交 |
| `__init__.py` | 保存包版本，并把命令行入口转发给 `cli.py` |

## 第一阶段目标

完成一个不包含工具调用的最小聊天闭环：

```text
用户输入
   ↓
cli.py 处理终端交互
   ↓
agent.py 组织最近的多轮消息
   ↓
llm.py 调用 DeepSeek 并返回流式文本
   ↓
cli.py 实时展示回答
   ↓
agent.py 在回答完整结束后保存本轮消息
```

### 当前配置

将 `.env.example` 复制为 `.env`，然后填写真实 API Key：

```dotenv
DEEPSEEK_API_KEY=your_real_api_key
DEEPSEEK_BASE_URL=https://api.deepseek.com/anthropic
DEEPSEEK_MODEL=deepseek-v4-flash
THINKING_ENABLED=false
SYSTEM_PROMPT="You are Neil Agent, a helpful local coding assistant. Give accurate, practical, and concise answers."
MAX_TOKENS=8192
MAX_ROUNDS=20
MAX_CONTEXT_CHARS=120000
MAX_TOOL_ROUNDS=5
REQUEST_TIMEOUT=120
MAX_RETRIES=2
RETRY_BASE_DELAY=1
RETRY_MAX_DELAY=8
WORKSPACE_ROOT=.
COMMAND_TIMEOUT=120
MAX_COMMAND_OUTPUT_CHARS=20000
```

`THINKING_ENABLED` 默认是 `false`，让响应更直接、流式展示更及时。设置为 `true` 后，普通请求和流式请求都会启用 DeepSeek 思考模式。`SYSTEM_PROMPT` 用于调整 Agent 的角色和回答方式，不再需要修改 Python 代码。

## 2026-07-17：主动质量验证和本地 Git 保存

### 修改后验证

- Agent 在文件写入成功后会明确提醒模型根据修改类型选择合适的 `run_quality_check`，而不是无理由运行全部检查。
- 内置工具工作流会追加到可配置系统提示词之后，不会替换用户在 `.env` 中设置的角色提示词。
- 命令结果统一包含 `Command`、`Working directory`、`Exit code` 和 `Output`，便于模型准确汇总执行情况。
- 最终回答必须说明尝试了哪些检查、退出码和关键结果；用户拒绝审批时必须说明检查未运行。

### 本地 Git 保存

- 新增 `git_stage`，只接受最多 50 个明确的工作区相对文件路径。
- 拒绝 `.`、目录、工作区外路径、受保护目录、`.env` 和常见私钥文件，不提供 `git add -A` 或任意 pathspec。
- 暂存预览同时展示当前状态、已暂存内容、工作区 diff 和未跟踪文本文件内容。
- 暂存预览包含完整文件状态生成的 `Change-ID`；文件或暂存区变化后必须重新批准。
- `git add` 可能运行仓库配置的 clean filter，因此预览会明确提示风险，并始终要求人工批准。
- 新增 `git_commit`，只接受不超过 200 字符的单行提交消息，并在执行前展示暂存区 diff；超长内容会截断，但 `Change-ID` 覆盖完整 diff。
- 本地提交禁用 Git hooks 和 GPG 签名，不支持 amend、空提交、任意参数或推送。
- 暂存完成后提醒模型先调用 `git_diff(staged=true)`；提交完成后明确告知模型不能声称已经推送。
- CLI 当前注册 10 个工具，其中写入、质量检查、暂存和提交共 5 个工具必须人工确认。
- 增加主动验证提示、路径边界、敏感文件、旧审批失效、空暂存区和安全提交参数测试。

## 2026-07-20：本地会话保存和恢复

### 版本化快照

- 新增 `session.py`，使用带 `version: 1` 的严格 JSON 快照保存完整成功对话、任务计划和最近一次质量检查。
- 快照只接受预定义字段，不读取或保存 `.env`、系统提示词、API Key 和其他环境变量。
- 会话保存在工作区的 `.neil-agent/sessions/`；该目录加入 `.gitignore`，并被文件工具和 Git 暂存工具视为受保护目录。
- 单个会话文件最大 25 MB；读取时验证文件大小、会话 ID、文件名、版本、时区、完整消息轮次和任务步骤顺序。
- 文件必须位于真实会话目录中，拒绝路径穿越和符号链接绕过。
- 保存使用同目录临时文件、刷新与 `fsync`、`os.replace` 原子替换；失败时保留旧快照并清理临时文件。

### CLI 会话生命周期

- 新增 `hooks.py`，提供 `before_model`、`after_model`、`before_tool`、`after_tool` 四个进程内阶段；只接受宿主注册的 Python callable，不解析或运行任意 shell 字符串。
- 前置模型 hook 可拒绝或附加请求专用上下文，前置工具 hook 可在操作前拒绝；后置阶段仅审计，不能改变控制流或附加上下文。
- 单 hook 上下文、累计上下文、拒绝原因和每阶段回调数量都有上限，并拒绝不允许的控制/格式字符；回调异常和返回类型错误统一进入 `HookError` 边界。
- 工具前置失败采用 fail-closed，不执行 handler；工具后置审计失败会明确说明操作可能已经完成，不伪装成回滚。

### Token 与子进程安全结论

- DeepSeek 官方目前给出英文字符约 `0.3 token/字符`、中文字符约 `0.6 token/字符` 的近似值，并提供离线 tokenizer 演示包；没有为聊天前精确计数设计的轻量服务端接口。
- 本地软预算改用上述字符比例，其他非 ASCII 字符继续保守计算；实际 token 与费用仍以 DeepSeek 响应 `usage` 为准，文档不再把估算描述为精确 tokenizer 结果。
- 新增 `docs/sandbox-assessment.md`，评估 Windows AppContainer/Windows Sandbox 和 Linux bubblewrap。当前应用层白名单不声称是 OS 沙箱；在 fail-closed 文件与网络隔离后端完成前不开放通用 shell。
- 修正 `run_quality_check` 工具 schema 漏列离线 `eval` 的既有问题，使模型可见枚举和执行白名单重新一致。
- 自动化覆盖一次性三种输出、只读工具集合、错误清洗、显式保存、CLI 参数路由、hook 拒绝/审计/上限和 DeepSeek 估算比例；当前结果为 155 项通过、1 项 Windows 条件跳过。当前本地检查不调用真实 API；现有 `neil-agent-eval` 真实回归入口仍保留双重显式成本确认。

## 2026-07-23：非交互协议 v2 与一次性写入审批

## 2026-07-24：审计诊断与跨进程安全轮转

### 内核文件锁与失败语义

- 新增稳定的 `.neil-agent/audit/events.lock` 锁锚点；审计文件大小检查、单备份轮转和追加在同一个跨进程临界区完成，避免多个 Neil Agent 进程同时轮转或写入。

- `/doctor` 新增“生命周期审计”检查：未启用时明确报告可选关闭状态；启用时报告路径、当前/备份大小、轮转上限、有效记录数、无效记录数和锁状态。
- 诊断不会创建审计目录或文件；它先做非阻塞锁探测，锁空闲时在锁内扫描当前与备份 JSONL，锁繁忙时只给出警告而不与活动写入竞争。

## 2026-07-24：高级可视化 Phase 0A 事件基础

### 版本化运行时事件

- 新增不可变 `RuntimeEvent` 与 `RuntimeMetadataItem`，固定版本 1，使用唯一事件 ID、稳定关联 ID、父事件 ID、时区时间、阶段、状态和有界元数据。

- 新增进程内 `EventBus`，每个观察者拥有独立的固定容量队列和守护线程；Agent 发布只执行非阻塞入队，不等待观察者。

- 观察者数量、单队列容量和待投递数量均有明确边界；显式 `flush()` / `close()` 支持超时，但 Agent 主链路从不调用等待方法。
- 事件总线是可选注入能力；交互 CLI、非交互协议和基础 `/cockpit` 默认不创建它，因此当前输出契约和无观察者执行路径保持不变。
